# Monitoring a swarm with JoinActors

This guide shows how to run an example application demonstrating the use of the [JoinActors library](https://github.com/a-y-man/join-actors) to monitor a swarm of [Actyx machines](https://www.npmjs.com/package/@actyx/machine-runner).


## Overview
The example application (illustrated in [Figure 1](#architecture) below) has two components (1) **a swarm of Actyx machines** and (2) a **monitor written using the JoinActors library** receiving all messages generated by the swarm.

The [*swarm*](https://doi.org/10.4230/LIPIcs.ECOOP.2023.15) models an automated car factory where robots to retrieve parts from a warehouse and assemble them into a finished car. The swarm contains a special *forwarding process* that receives all messages generated by the swarm and forwards them to the monitor. The forwarding process has been automatically generated for the example using the tool [`smcli`](https://github.com/DTU-SSE/swarm-monitor).

The monitor uses [*join patterns*](https://doi.org/10.4230/LIPIcs.ECOOP.2024.17) to look for certain combinations of messages received from the forwarder. Specifically, the monitor looks for four different message combinations, respectively indicating that:
1. A part has been requested and successfully delivered from the warehouse.
2. A part has been requested and delivered from the warehouse, but is found to be defectful.
3. A car has been built.
4. The warehouse has closed.

## Getting and running the example
The example application is available as [a Docker image](https://hub.docker.com/r/lucas8744/swarm-monitor) on Docker Hub. The four steps below shows how to fetch the image and run the application.

#### 1. Getting the image
To download the image, please open a terminal and run:
```bash
docker pull lucas8744/swarm-monitor
```

#### 2. Setting up a Docker network
In order for the swarm and the monitor to be able to communicate, we need to set up a Docker network. To set up a Docker network, please run:

```bash
docker network create swarmnet
```

This sets up a network called `swarmnet` over which the swarm and the monitor can communicate.

#### 3. Starting the monitor
To start the monitor, please run:
```bash
docker run -it --rm --name monitor --network swarmnet lucas8744/swarm-monitor monitor
```

#### 4. Starting the swarm
To start the swarm, please **open a new terminal** and run:

```bash
docker run --rm --network swarmnet -it lucas8744/swarm-monitor swarm -a monitor
```
